Hackathon Field Guide A starting point for hackers
1
What you will need basic supplies
You will need only a few basic things to get hacking:
  • At least 1 computer
  • At most 3 other teammates
  • 24 hours
  • Flexibility
  • Tenacity

Hacking is hard work!

It can be emotional. It can be frustrating. Teams that do well are ones that know how to get UNSTUCK. If you find yourself getting stuck on something:
  • Ask for help
  • Cut the feature you were working on
  • HACK! Find a way _around_ what you're stuck on
Other than this, the only thing you need (as in life) is to

have fun

2
What you will build ideas
You should build whatever you want. Seriously...

Whatever you want

One of the best ways to get started is to find something cool and then play with it and make something even cooler out of it. It can be anything:
  • A dataset
  • A really cool open source library
  • A powerful new API
or just make that series of bash scripts that you've been meaning to for a while now. You can build anything

(Please have it involve technology though ^_^)

Some examples:
  • A chrome extension that previews as you scroll
  • A Facebook skin that looks exactly like Google
  • A game that tests your knowledge of your friends
have all won Facebook hackathons in the past.

See chapter 4 for more about the judging criteria

3
How you will build tools
There are a lots of things that you can build. The most common are:
  1. Mobile Apps
  2. Web Apps
  3. Facebook Apps
  4. Games
If you want to build one of these, but don't know where to begin, you've come to the right place!
3.1
Tools for Mobile Apps~ iphone android windows
To get started you'll need the right tools. Please do all of the following in order:
  • Android: IDE -> ADT -> SDK
  • iPhone: IDE -> SDK
  • Windows Phone: IDE -> SDK -> More steps
or you can consider using PhoneGap to write your app in JS and html. This is especially useful if people on your team have different devices.
Backend
If your mobile app will need to store data in a persistant way, particularly over web, it is recommended that you use Parse. Parse gives you amazing SDKs for iPhone, Android, and Windows Phone. See their tutorials to get started.
3.2
Tools for Web Apps~ php ruby js python

At Facebook Hackathons we ask that you host your apps somewhere so that other people can use your hack! You have a couple options, depending on what language you want to write your server logic in. The recommended services are:

  • Heroku - For PHP, Python, Java and Ruby servers. See Dave Mercurio's excellent guide for getting started with Heroku
  • Parse - For purely JS apps (using express or node.js)
If you've never done web programming before, the examples here (PHP) will give you some idea how to get started.
3.3
Tools for Facebook Apps the social web
We love to see people make apps that integrate with Facebook! The first step to making a Facebook app is just to make a normal web app (see above). After you've gotten your web app going, register you Facebook App here and checkout our documentation then start playing with the explorer.

Facebook has SDKs for:

  • iOS
  • Android
  • The web (including PHP and JS)
  • Unity
If you're making a Facebook canvas app, this boilerplate PHP should help you get started, as will this tutorial.
3.4
Tools for Games~~~ html5 canvas mobile
Some of the coolest hackathon projects are games. Thankfully people have built some amazing platforms for you to get started quickly. I highly recommend using Unity. Not only does Unity work across mobile, native and web, it also works amazingly well with Facebook for social features and with Parse to make persisting data trivially easy.
4
How you win~~ objectives
As said above, the main point is to have fun and learn something new. That said, you win by also making something cool. Judging is kind of subjective but is 50/50:
  • Idea: Something exciting we've never seen before. Something that sounds useful, fun or engaging. Something people want.
  • Execution: Built really well. Acutally does what it set out to do: no more, no less. Has a useful interface, is hosted somewhere where the judges can play with it, and is technically impressive.
